## Розділ 0. Вступ (текст для респондента)

> Ми створюємо open-source MCP-сервер над порталом відкритих даних України (`data.gov.ua`). MCP-сервер — це проміжний шар, що дозволяє AI-помічникам (Claude, ChatGPT, локальні LLM) звертатись до порталу природною мовою.

> Ваші відповіді допоможуть нам сфокусуватись на тих інструментах, що дійсно потрібні користувачам. Опитування анонімне, заповнення ~10 хв. Результати ми опублікуємо у репозиторії проєкту.

## Розділ 4. Формат відповідей

> **Технічна довідка:** MCP-tool повертає структуровані дані до LLM. **Те, як LLM розповість вам результат** (таблиця, переказ, мова) — залежить від **вашого** prompt'a, не від MCP. Питання нижче — лише про те, що ми **дійсно можемо контролювати** на стороні MCP: що класти в tool-result, як обробляти великі обсяги, які додаткові поля включати.

### 4.1. Коли у ресурсі **тисячі/мільйони рядків** — як MCP має поводитись? `[single-choice]`
> *(великий tool-output швидко заповнить контекстне вікно і LLM втратить здатність нормально відповідати. Шукаємо ваш баланс)*
- Завжди повертати лише top-N (наприклад 20) + сигнал «є ще, уточніть фільтр»
- Top-N + посилання на файл/SQL-запит для повного скачування **поза** LLM-сесією
- Дати LLM можливість запросити ще через пагінацію (наступна сторінка)
- Зберегти повний результат як artifact / файл, в чат повернути лише summary + шлях
- Не важливо — довіряю інструменту вирішити

### 4.2. Чи має tool-result містити **цитату-посилання** на оригінальний датасет/ресурс? `[single-choice]`
- Так, завжди (для верифікації / журналістики)
- Так, опційно (через прапорець у виклику)
- Не важливо

### 4.3. Який спосіб виносити великі результати за межі контексту вам зручніший? `[multi-choice]`
- Зберегти у локальний файл (`./output/result.csv`)
- Повернути URL для download з hosted instance
- Скопіювати у clipboard
- Згенерувати тимчасове посилання на оригінальний `data.gov.ua` ресурс
- Не потребую — працюю лише з малими вибірками

### 4.4. Які **додаткові поля-збагачення** ви хотіли б отримувати в tool-result? `[multi-choice]`
> *(MCP може дозбагатити дані з порталу — наприклад, перекласти UUID-slug у людинозрозумілу назву, додати URL ліцензії, нормалізувати дати)*
- Людинозрозумілий URL датасету (замість UUID-slug — title)
- URL ліцензії та її версія (cc-by → https://creativecommons.org/licenses/by/4.0/)
- Англомовний переклад title/notes (для en-LLM-агентів)
- Англомовний переклад тегів (auto-translate UA→EN)
- Health-check статус ресурсу (URL живий / broken)
- Розмір ресурсу у людинозрозумілих одиницях (MB замість bytes)
- Дата останнього оновлення у відносному форматі («2 дні тому»)
- Нічого з переліку — потрібні лише сирі дані як у CKAN
- Інше: __________

### 4.5. Мова **технічних полів** у tool-result (назви полів, помилки, статусні повідомлення MCP) `[single-choice]`
> *(дані з порталу повертаються as-is. Це питання — про метаповідомлення самого MCP-сервера)*
- Українською (`помилка`, `опис`, `кількість`)
